Database Management System

Data

Data refers to raw facts and figures that by themselves may not carry any specific meaning. It could be numbers, characters, symbols, or even audio and video.

Information

Information is processed data that has meaning and is useful for decision-making. When data is put into context or organized in a meaningful way, it becomes information.

Features of Information

  • Accurate
  • Relevant
  • Timely
  • Complete
  • Understandable
  • Verifiable

Database and Its Purpose

A database is an organized collection of structured data, typically stored electronically. It enables users to store, retrieve, and manipulate data efficiently.

Terminologies in Database

  • Table: Collection of related data in rows and columns.
  • Field: Column representing a data attribute.
  • Record/Tuple: A row representing a data entry.
  • Object: Data + methods (object-oriented).
  • Key: Uniquely identifies a record.

Data Dictionary

A data dictionary stores definitions and descriptions of data elements. It helps maintain integrity and consistency in a database.

Database Management System

Introduction

A DBMS is software that manages databases and provides an interface to interact with the data.

Objectives

  • Efficient data access
  • Ensure data integrity and security
  • Support multiple users

Advantages

  • Better data sharing and security
  • Reduced redundancy
  • Backup and recovery support

Disadvantages

  • Complex and costly setup
  • Needs skilled personnel

Types of Database Model

  • Hierarchical: Tree structure with single-parent records.
  • Network: Many-to-many relationships via links.
  • Relational: Tabular format with SQL.
  • Entity-Relational: Conceptual model using entities and relations.

Integrity Constraints and Types

  • Domain: Value restrictions per field.
  • Entity: Primary key must be unique and not null.
  • Referential: Foreign key must match primary key.
  • Keys: Primary, Foreign, Candidate, Composite.

Normalization

Introduction

Normalization reduces redundancy and improves data structure.

Normal Forms

  • 1NF: Atomic fields
  • 2NF: No partial dependencies
  • 3NF: No transitive dependencies

Advantages

  • Less redundancy
  • Better consistency

Disadvantages

  • Complex queries due to joins

Centralized and Distributed Database

Introduction

Centralized DB is stored in one place; distributed DB is spread across networks.

Advantages of Centralized

  • Easy to manage
  • Better control

Disadvantages of Centralized

  • Single point of failure

Advantages of Distributed

  • Faster access
  • More reliable

Disadvantages of Distributed

  • Complex setup

Comparison Table

Feature Centralized Distributed
Speed Slower Faster
Failure Impact High Low
Maintenance Simple Complex

Database Security

Introduction

Database security protects information from unauthorized access, loss, or corruption.

Challenges

  • Hacking
  • Insider misuse
  • Lack of encryption

Security Measures

  • User authentication
  • Access controls
  • Data encryption
  • Firewalls
  • Regular backups

Roles of DBA

  • Database setup
  • Monitoring performance
  • Managing users and roles
  • Security enforcement
  • Backup and restore
